AI-Assisted Design Workflow

Three hands-on exercises to go from idea to pixel-perfect design using Claude, HTML, and Figma.

Exercise 1

Ideate, Concept & Wireframe

1
Open a new Claude Chat Choose Opus 4.6 + extended thinking
▼

Go to ↗ claude.ai and open a new chat.

In the model selector, choose Claude Opus 4.6 and enable Extended Thinking for deeper reasoning on complex design challenges.
2
Ideation — Generate HMW answers How Might We prompt for patient overview
▼

Copy and paste this prompt into Claude:

Prompt to copy
Create 20 answers to this question: How might we create an overview of a patient across the primary and secondary health sector and healthcare professionals in Denmark
3
Konceptualize — Describe a digital product Turn HMW answers into a product concept
▼

In the same chat, follow up with:

Prompt to copy
Describe a digital product for a general practitioner based on the HMW answers
4
Flow — Create a user journey Map how the GP uses the product
▼

Continue in the same chat:

Prompt to copy
Use the concept description to create a user journey
5
Paste to Figma — Share the user journey Screenshot it into the Figma output file
▼

Take a screenshot of Claude's user journey output and paste it into the shared Figma Design file.

↗ Open Figma Output File
6
Wireframe — 3 essential desktop screens Generate HTML wireframes
▼

Ask Claude to wireframe the product:

Prompt to copy
Wireframe 3 most essential desktop screens of the product

After Claude responds, ask it to turn each screen into a separate file:

Follow-up prompt
Create a separate HTML file for each of the three wireframes
7
Download the 3 HTML files Save them locally for the next exercise
▼

Click the download icon on each of the 3 HTML artifacts that Claude generated. Keep them in an easy-to-find folder — you'll need them in Exercise 2.

Tip: Name them screen-1.html, screen-2.html, screen-3.html so they're easy to identify.
Exercise 2

HTML → Figma Make

8
Open a new Figma Make file Use Figma's AI design tool
▼

In Figma, create a new Figma Make file from the project dashboard. This is where you'll import each of your HTML wireframes.

↗ Open Figma Project
9
Convert HTML files using Figma Make Upload one at a time with Opus 4.6
▼

In Figma Make, select Claude Opus 4.6 as the model. Upload each HTML file one at a time and use this prompt:

Prompt to copy
Create a Figma design file from this HTML
Repeat for all 3 HTML screens, one at a time.
10
Spend 5 min adjusting in Figma Clean up the generated frames
▼

Review the 3 generated frames. Make any quick adjustments — align elements, rename layers, tweak spacing. Don't overthink it — this is a 5-minute pass.

Exercise 3

Pixel-Perfect Design with Design System

11
Open the Claude Chat Project Design system files are in Project Knowledge
▼

Open the shared Claude project — it already has the design system files loaded in Project Knowledge.

↗ Open Claude Project
Tip: When referencing a Figma frame, paste the Figma selection link (right-click a frame → Copy link) directly into your message for more context.
12
Prompt Claude to create a pixel-perfect design Opus 4.6 + extended thinking
▼

Switch to Claude Opus 4.6 and enable Extended Thinking. Upload your wireframe HTML files and send this prompt:

Prompt to copy
Use wireframes to create a pixel perfect design using the design system files in the project knowledge
13
Open and review the new designs Check the styled output from Claude
▼

Preview the HTML files Claude generated. Compare them visually against your original wireframes — you should see the design system styles applied.

14
Paste to Figma — Before & After Share screenshots in the Figma output file
▼

Take screenshots of both your wireframes and the final styled designs. Paste them into the shared Figma Design file and label them Before and After.

↗ Open Figma Output File
What to try next

Bonus & Exploration

★
Try Claude Cowork Ask Claude to suggest use cases
▼

In the Claude desktop app, switch to Cowork mode and ask:

Prompt to copy
List some use-cases I should try in Claude Cowork for a design workflow
★
Install Figma plugin in Claude Code Try the "Send to Figma" feature
▼

In Claude Code, set up the Figma MCP connector. Then ask Claude to generate a component and use "Send to Figma" to push it directly into your design file.

Look for the Figma plugin in Claude Code settings under MCP Servers.
★
Commit and push to GitHub Save your work
▼

Stage, commit and push your HTML files to your GitHub repo:

Terminal commands
git add .
git commit -m "Add wireframes and exercise outputs"
git push -u origin main
Trouble? Ask Claude what's wrong and paste a screenshot for more context.
